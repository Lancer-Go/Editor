一、需求背景

1.1 现状问题

随着 AI 编码助手（如 Cursor、Antigravity、Claude）的普及，产品经理（PM）越来越多地使用 AI 直接生成 HTML 高保真原型。然而，AI 产出的原型通常需要微调——修改文案、调整布局、增删组件——而 PM 通常不具备前端编码能力。

核心矛盾

"AI 能快速生成原型，但 PM 无法快速修改原型"

  • 让 AI 反复修改效率低，一个文案改动需要重新描述需求、等待生成
  • 手写 HTML/CSS 对 PM 门槛太高
  • 墨刀、Axure 等原型工具无法直接编辑 AI 产出的 HTML 文件

1.2 为什么现在做

  • AI 原型工作流正在被越来越多 PM 采用，"AI 生成 + 人工微调"的需求迫切
  • 现有项目已有一个可用的雏形，具备基本编辑能力，有迭代基础
  • 市面上没有专门面向"编辑 AI 生成 HTML 原型"场景的轻量级工具

二、需求目标

目标类型描述衡量指标目标值
核心体验 PM 能像用墨刀一样拖拽组件编辑 HTML 原型 组件添加到画布的操作步骤 ≤ 2 步（拖 + 放）
效率提升 原型微调不再依赖 AI 反复修改 单次文案/布局微调耗时 ≤ 30 秒
交付能力 原型可直接导出用于 PRD 写作和存档 批量导出完成时间 ≤ 10 秒 / 页
开源价值 项目开源，吸引社区参与完善 代码可读性 & 架构清晰度 新贡献者 30 分钟内能理解核心结构

三、用户与使用场景

3.1 典型用户画像

👤 小李 — 初级产品经理

  • 背景：2 年工作经验，会使用墨刀画原型，最近开始使用 AI 工具辅助工作
  • 技术能力：不会写代码，能看懂简单的 HTML 结构
  • 工作流：用 AI（Cursor / Antigravity）生成 HTML 原型 → 需要微调文案和布局 → 截图嵌入 PRD
  • 痛点：AI 生成的原型总是差那么一点，让 AI 改来改去不如自己动手，但自己不会写代码

交互流程图

原型预览：组件面板

交互逻辑

  1. 左侧文件浏览面板上方新增「📦 组件」Tab，与「📂 文件」Tab 切换
  2. 组件面板按类别分组折叠展示（基础 / 表单 / 按钮 / 容器 / 导航 / 反馈）
  3. 用户鼠标按住组件缩略图 → 拖拽至画布 → 释放时在鼠标位置插入对应 HTML 元素
  4. 插入后自动选中该元素，右侧属性面板显示其属性

组件模板定义

每个组件对应一段预设的 HTML 片段。组件清单：

📝 基础元素
  • 文本段落 <p>
  • 标题 <h1> ~ <h6>
  • 分割线 <hr>
  • 图片占位 <div> + 占位图标
  • 图标 <span> + SVG
📋 表单控件
  • 输入框 <input type="text">
  • 下拉选择 <select>
  • 复选框 <input type="checkbox">
  • 单选框 <input type="radio">
  • 开关 Toggle
  • 搜索框 <input type="search">
  • 日期选择器 <input type="date">
🔘 按钮类
  • 主按钮（填充样式）
  • 次按钮（描边样式）
  • 文字链接
  • 图标按钮
📦 容器类
  • 卡片容器
  • 弹窗 / 模态框
  • 选项卡 (Tab)
  • 折叠面板 (Accordion)
  • 数据表格
🧭 导航类
  • 顶部导航栏
  • 侧边菜单
  • 面包屑
  • 分页器
💬 反馈类
  • Toast 消息提示
  • 进度条
  • 徽标 (Badge)
  • 标签 (Tag)

业务规则

  • 组件拖入画布时，默认采用 position: relative，让元素参与正常文档流
  • 如果画布区域不可见（如 iframe 未加载），拖入时给出提示：「请先打开一个 HTML 文件」
  • 每个组件携带预设的 CSS 样式，确保拖入即美观
  • 支持 Ctrl+Z 撤销刚插入的组件

5.2 智能对齐辅助线

交互流程图

flowchart TD
  A["mousedown 开始拖动"] --> B["计算拖拽元素六条线"]
  B --> C["遍历同级可见元素"]
  C --> D{"任意线距 ≤ 5px?"}
  D -- 是 --> E["显示红色辅助线 + 吸附"]
  D -- 否 --> F["隐藏辅助线"]
  E --> G{"mouseup?"}
  F --> G
  G -- 否 --> B
  G -- 是 --> H["隐藏辅助线 + pushUndo"]

原型预览：画布区域

交互逻辑

  1. 拖拽移动元素时，实时计算与画布内其他可见元素的距离
  2. 当边缘或中心线对齐时（阈值 ≤ 5px），显示红色辅助线并自动吸附
  3. 对齐类型：
    • 左边缘对齐、右边缘对齐、顶边对齐、底边对齐
    • 水平中心对齐、垂直中心对齐
    • 等距分布辅助线（多个元素间距相等时提示）
  4. 松开鼠标后辅助线消失

业务规则

  • 辅助线颜色：■ 红色实线（1px），确保在白/灰背景上醒目
  • 吸附阈值：5px，超过则不吸附
  • 性能：仅检查画布中可见的同层级元素（避免深层嵌套元素干扰）

5.3 项目级多页面管理

交互逻辑

  1. 将同一文件夹中的所有 HTML 文件视为一个「项目」
  2. 文件浏览器中展示项目视图：文件夹名即项目名，下属 HTML 文件为项目页面
  3. 页面间跳转链接配置：
    • 选中一个按钮或链接元素 → 右侧面板出现「链接到」配置项
    • 下拉列表显示同项目内的其他 HTML 页面
    • 选择后自动设置元素的 href 属性指向目标页面文件名

5.4 文件浏览优化

优化项实现方式
🔴 修复空文件夹 Bug 空文件夹时仍显示「.. 返回上级」入口
记忆上次目录 将当前目录存入 localStorage，下次启动自动填充
最近文件列表 记录最近打开的 10 个文件路径，展示在文件面板顶部
拖拽文件夹打开 支持将系统文件夹拖入编辑器窗口，自动识别路径并打开

5.5 批量导出图片/PDF

交互逻辑

  1. 工具栏新增「📤 导出」按钮
  2. 点击弹出导出面板：
    • 单页导出：导出当前打开的页面
    • 批量导出：勾选项目中的多个页面
    • 格式选择：PNG（默认）/ PDF
  3. 后端使用 Puppeteer 或类似方案打开 HTML 并截图/生成 PDF
  4. 完成后提供下载链接或直接保存到项目 exports/ 目录

5.6 源码编辑模式

交互逻辑

  1. 工具栏新增「</>」源码切换按钮
  2. 点击后画布区域切换为代码编辑器视图（左右分栏 或 全屏代码）
  3. 特性：
    • HTML / CSS / JS 语法高亮
    • 源码修改实时反映到可视化预览（双向同步）
    • 可使用 CodeMirror 或 Monaco Editor 方案
  4. 再次点击切换按钮 → 回到可视化编辑模式

七、异常与边界处理

场景处理方式
画布未加载文件时拖入组件Toast 提示「请先打开一个 HTML 文件」，阻止操作
拖拽组件到画布外释放取消插入，无任何副作用
打开非 HTML 文件文件列表中仅展示 .html/.htm 文件，不可选非 HTML
文件被外部修改再次打开时以磁盘最新版本为准，提示「文件已更新」
保存失败（磁盘满/无权限）Toast 错误提示 + 保留编辑内容在 iframe 中不丢失
目录不存在 / 路径错误提示「目录不存在」+ 建议检查路径
空文件夹显示「此目录下没有 HTML 文件」+ 保留「返回上级」按钮
超大 HTML 文件（>5MB）正常加载但提示「大文件编辑可能较慢」
导出失败（Puppeteer 未安装）提示安装依赖命令 npm install
